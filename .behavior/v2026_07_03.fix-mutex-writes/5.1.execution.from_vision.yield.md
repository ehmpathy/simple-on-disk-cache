## contract (approved vision)

```ts
// default: source-first (correct)
createCache({ directory });

// opt-in: memory-first (fast)
createCache({ directory, consistency: 'memory-first' });

// per-read override on a memory-first cache
await cache.get(key, { consistency: 'source-first' });
```

## design notes

- effective read consistency = `options.consistency ?? cacheDefaultConsistency ?? 'source-first'`
- memory is used (read + write) only when the cache default is `memory-first`
- a source-first *read* on a memory-first cache repopulates memory (keeps it warm), per wish acceptance
- cross-process overwrite simulated in tests via a second `createCache` instance (separate memory closure) that writes the same key to the same directory
