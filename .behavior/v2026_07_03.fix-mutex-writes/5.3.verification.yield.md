## defects found & fixed in verification

### flake: cloud item-level expiration test

- **symptom**: `cache › cloud › should respect the item level expiration for the cache` failed once (24/25) — the "still valid" check returned `undefined` instead of `'solid'`.
- **root cause**: the polarity flip exposed a latent flake. under the old memory-first default, the post-set gets hit memory (instant); under the new source-first default, every get hits S3 (~2-4s each). the test used a 5s TTL with checks at 3s/6s — margins (1-2s) smaller than an S3 op. accumulated read latency drifted the "valid at 3s" check past the 5s TTL boundary, so the value read as expired.
- **fix**: widened the cloud test's windows to absorb S3 latency — 12s TTL, checks at 5s and 16s (from `genTimer` start). preserves the test intent (valid before TTL, expired after) with margins (~5s valid-side, ~4s expired-side) that exceed worst-case S3 latency. local version left unchanged (instant ops, never flaked).
- **verified**: re-ran integration → 25 passed, 0 failed (77s).
