### acceptance coverage (blackbox via public contract)

`src/cache.conditionals.acceptance.test.ts` proves the wish's headline journeys **blackbox via
the public package entry** (`./index` — `createCache` / `isSimpleCacheConditionError` /
`SimpleCacheConditionError` / `DirectoryToPersistTo` / `SimpleOnDiskCacheCloudAdapter`, no
reach-in to internals), per `rule.require.acceptance.blackbox`:
- **base surface (plain get/set/keys)** — the non-conditional public SDK path: set → get
  round-trip, last-writer-wins overwrite, absent-key read (`undefined`), and invalidation that
  drops the key from `keys()` (a one-key list → the empty `""` list). added in peer i007 r9 to
  close the base-contract acceptance gap per `rule.require.test-coverage-by-grain`.
- **compute-once / stampede** — two racers put-if-absent; the loser catches
  `SimpleCacheConditionError` (via the exported predicate) and converges on the winner's value —
  the vision's literal "aha moment" catch → re-read narrative.
- **distributed-lock lifecycle** — acquire (put-if-absent) → contend (rejected) → **stale-token
  release rejected (lock stays held)** → release (compare-and-delete) → reacquire; plus a
  **ttl-reclaim** step and a **stale-lock auto-reclaim** step (a planted 60s-old lock is
  reclaimed, which honors expiry).
- **optimistic concurrency** — a stale-token compare-and-set is rejected; the fresh value
  survives (no lost update).
- **conditional read** — `get(key, { condition })` returns the value on a matched token and
  rejects a stale one (mismatch first-line snap).
- **version reader** — `version(key)` reports `undefined` for an absent key and an opaque token
  for a present one.
- **cloud tier (real s3)** — a global compute-once journey (put-if-absent race, exactly one wins,
  the loser converges), a deterministic global compute-once loser (sequential, so the conflict
  message is pinned), and the full global-lock lifecycle (acquire → contend → stale-release-rejected
  → release → reacquire, all five stages snapshot-pinned symmetrically with the local twin) run
  through the public contract against the real test bucket, with `via: sdkAwsS3` (the REAL adapter,
  no fake) — added
  in peer i006 r9 (journeys) and i007 r9 (snapshot symmetry) so the vision's "s3 = global" headline
  usecase is proven at the acceptance gate, not deferred to integration on a credential-difficulty
  excuse (`rule.require.acceptance-journey-coverage`).
the local disk journeys are deterministic + credential-free; the cloud journey needs aws creds and
is proven against real s3 here AND by the integration suite. the non-conformant-cloud-adapter
friction (a custom `via` that would silently downgrade a conditional write) is a fail-loud guard
proven in the INTEGRATION suite — it needs a hand-crafted fake adapter, which an acceptance test
must not substitute for a real dependency (`rule.forbid.acceptance.mocks`).
