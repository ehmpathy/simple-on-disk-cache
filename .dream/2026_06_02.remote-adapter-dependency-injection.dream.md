dream = remove @aws-sdk dependency via remote adapter injection

## .what

replace hardcoded S3 client with a user-supplied remote adapter interface

## .why

- `@aws-sdk/client-s3` has internal cyclical deps (excluded in .dpdmrc.yaml)
- heavy dependency for a "simple" cache library
- couples package to AWS when users may want GCS/Azure/etc

## .how

```ts
type RemoteAdapter = {
  get: (key: string) => Promise<string | null>;
  set: (key: string, value: string) => Promise<void>;
};

// user brings their own adapter
createCache({
  directory: {
    remote: {
      get: async (key) => /* user's S3/GCS/whatever impl */,
      set: async (key, value) => /* user's impl */,
    }
  },
})
```

## .options considered

1. **dependency injection** (recommended) — users bring own adapter
2. **separate adapter package** — `simple-on-disk-cache-s3`
3. **peer dep + lazy import** — still couples interface to S3

## .migration

- deprecate `{ s3: { bucket, prefix } }` config
- export current S3 utils as convenience (optional dep)
- new interface: `{ remote: RemoteAdapter }`
