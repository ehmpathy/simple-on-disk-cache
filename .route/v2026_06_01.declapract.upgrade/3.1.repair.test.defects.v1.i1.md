## defects repaired in stone 2

these defects were identified and fixed in stone 2.detect.hazards:

### defect 1: yq syntax incompatibility

- **what**: test:lint:cycles fails with yq lexer error
- **how**: declapract template used jq-style syntax incompatible with Go yq
- **why**: Go version of yq (mikefarah/yq) has different expression syntax than jq-wrapper yq
- **fix**: changed `if length > 0 then join("|") else "^$" end` to `join("|") // "^$"` in package.json

### defect 2: dpdm third-party cycles

- **what**: test:lint:cycles fails on node_modules cycles
- **how**: dpdm follows import chains into node_modules
- **why**: third-party packages (bottleneck, @aws-crypto, @aws-sdk) have internal cycles
- **fix**: added `node_modules` to .dpdmrc.yaml exclude list

### defect 3: depcheck false positives

- **what**: test:lint:deps reports unused devDependencies
- **how**: depcheck cannot detect dynamic imports
- **why**: rhachet-roles-rhachet and test-fns are loaded indirectly
- **fix**: added packages to .depcheckrc.yml ignores

### defect 4: provision type error

- **what**: TypeScript error TS2339 on pkg.private access
- **how**: package.json import type lacks optional private field
- **why**: TypeScript infers narrow type from actual file contents
- **fix**: cast pkg to wider type with optional private field

## non-defect: s3 test failures

the 6 failed s3 integration tests are NOT code defects:

```
CredentialsProviderError: Your session has expired. Please reauthenticate.
```

- **what**: s3 tests fail with credential expiration error
- **why**: AWS SSO session has expired
- **why not a defect**:
  - error is from AWS credential provider, not application code
  - mounted tests (12/12) pass; this verifies cache logic works
  - s3 tests would pass with fresh credentials
- **resolution**: human must re-authenticate with AWS SSO
