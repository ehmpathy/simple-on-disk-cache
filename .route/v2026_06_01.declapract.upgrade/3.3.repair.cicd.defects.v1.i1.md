# 3.3.repair.cicd.defects

## purpose

document and fix CICD defects discovered after the declapract upgrade push.

---

## defect 1: aws auth mechanism changed

**what:** S3 integration tests fail with "Region is absent" in CI

**how:** declapract upgrade changed `.github/workflows/test.yml` and `.test.yml` from secrets-based AWS auth to OIDC-based auth

before (origin/main):
```yaml
# test.yml
secrets:
  aws-account-id: ${{ secrets.DEV_AWS_ACCOUNT_ID }}
  aws-access-key-id: ${{ secrets.DEV_AWS_ACCESS_KEY_ID }}
  aws-secret-access-key: ${{ secrets.DEV_AWS_SECRET_ACCESS_KEY }}
```

after (upgrade):
```yaml
# test.yml
with:
  creds-aws-region: us-east-1
  creds-aws-role-arn: ${{ vars.CREDS_CICD_AWS_DEV_OIDC_ROLE_ARN }}
```

**why:** the repo has the old secrets but not the new OIDC role ARN variable. the workflow skips AWS auth when the variable is absent, so S3 tests fail.

**fix:**
- set `CREDS_CICD_AWS_PREP_OIDC_ROLE_ARN` variable in GitHub repo settings
- updated test.yml to reference the new variable name (PREP instead of DEV)

## defect 2: case-sensitive error match for S3 NoSuchKey

**what:** S3 integration tests fail with `could not find object in s3 in bucket '...' with key '...valid_keys'`

**how:** `getObjectAsString.ts` throws error with lowercase message, but `cache.ts` catches with uppercase check

```typescript
// getObjectAsString.ts:20 - throws lowercase
throw new Error(`could not find object in s3 in bucket '${bucket}' with key '${key}'`);

// cache.ts:114 - catches uppercase (doesn't match!)
if (error.message.includes('Could not find object in s3 in bucket'))
```

**why:** `.includes()` is case-sensitive. the catch never matched, so `NoSuchKey` errors propagated up instead of a return of `undefined`.

**note:** this bug existed on main but was masked because prior CI runs had already created the `valid_keys` file in S3. the OIDC change exposed it via a fresh test environment.

**fix:** changed `cache.ts:114` to use lowercase `'could not find object in s3 in bucket'`
