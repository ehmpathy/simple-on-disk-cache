## test hazards

### t1. jest reporter dependency

**what**: jest configs added `test-fns/slowtest.reporter.jest` reporter
**why may break**: if test-fns not installed or version incompatible, jest fails to start
**verify**: `rhx git.repo.test --what unit --mode apply`

### t2. module name mapper

**what**: jest added `moduleNameMapper` for `@src/*` alias
**why may break**: must match tsconfig paths; mismatch causes import resolution failures
**verify**: `rhx git.repo.test --what unit --mode apply`

### t3. @jest/globals import

**what**: jest env files now use `import { jest } from '@jest/globals'` instead of global jest
**why may break**: requires @jest/globals package; old jest versions may not work
**verify**: `rhx git.repo.test --what unit --mode apply`

### t4. keyrack integration

**what**: jest env files now call `keyrack.source()` to inject credentials
**why may break**: if keyrack.yml not configured or keyrack locked, tests fail to start
**verify**: run tests locally without keyrack configured

### t5. env var name changes

**what**: `STAGE` → `CONFIG`, `I_KNOW_WHAT_IM_DOING` → `I_KNOW_THE_RISKS`
**why may break**: CI or local setups that set old env vars will not work
**verify**: search codebase and CI for old env var names

## cicd hazards

### c1. dpdm cycle detection

**what**: new `test:lint:cycles` command checks circular dependencies via dpdm
**why may break**: pre-existent cycles in origin/main will now fail lint
**verify**: `npm run test:lint:cycles`

### c2. stricter test commands

**what**: test commands now use `set -eu` — exit on error, exit on unset var
**why may break**: any unset env var or non-zero exit code fails the command
**verify**: `npm run test`

### c3. node version upgrade

**what**: `.nvmrc` changed from `v20.12.2` → `v22.21.0`
**why may break**: node 22 has different behavior than node 20; may have broken APIs
**verify**: `rhx git.repo.test --what unit --mode apply`

### c4. aws auth changed to oidc

**what**: test workflow removed explicit AWS secrets, now uses OIDC role ARN via `vars.CREDS_CICD_AWS_DEV_OIDC_ROLE_ARN`
**why may break**: if OIDC role not configured in github repo vars, AWS auth fails
**verify**: check github repo settings for CREDS_CICD_AWS_DEV_OIDC_ROLE_ARN var

### c5. pnpm install now runs prepare

**what**: `.install.yml` removed `--ignore-scripts` from `pnpm install`
**why may break**: prepare hooks (husky, rhachet init) now run in CI; may fail or add overhead
**verify**: check CI logs for prepare hook output

## config hazards

### cfg1. tsconfig module type

**what**: changed `module: "commonjs"` → `module: "node16"`
**why may break**: node16 uses native ESM semantics; may affect dynamic imports or module resolution
**verify**: `rhx git.repo.test --what types`

### cfg2. tsconfig module resolution

**what**: changed `moduleResolution: "node"` → `moduleResolution: "node16"`
**why may break**: node16 requires explicit file extensions in some cases
**verify**: `rhx git.repo.test --what types`

### cfg3. biome includes expanded

**what**: biome now includes `provision/**/*.ts` in lint scope
**why may break**: provision code may have lint violations
**verify**: `rhx git.repo.test --what lint`

### cfg4. changelog.md deleted

**what**: `changelog.md` file deleted
**why may break**: some release tools expect changelog.md; docs may reference it
**verify**: check if any workflows or docs reference changelog.md
