## defect 1: aws auth mechanism changed

### classification: repo quirk

### rationale

the declapract practice uses `CREDS_CICD_AWS_DEV_OIDC_ROLE_ARN` as the variable name:

```yaml
# from declapract-typescript-ehmpathy/src/practices/cicd-common/best-practice/.github/workflows/test.yml
creds-aws-role-arn: ${{ vars.CREDS_CICD_AWS_DEV_OIDC_ROLE_ARN }}
```

this repo uses `PREP` instead of `DEV` because:
- the repo has a prep environment, not a dev environment
- the variable was set up as `CREDS_CICD_AWS_PREP_OIDC_ROLE_ARN`

the practice is correct — it provides a sensible default. this repo's deviation from the default name scheme is a repo quirk.

### citation

```sh
gh search code "CREDS_CICD_AWS_DEV_OIDC_ROLE_ARN" --repo ehmpathy/declapract-typescript-ehmpathy --json path
```

found in:
- `src/practices/cicd-common/best-practice/.github/workflows/test.yml`
- `src/practices/cicd-package/best-practice/.github/workflows/publish.yml`
- `src/practices/cicd-service/best-practice/.github/workflows/deploy.yml`
