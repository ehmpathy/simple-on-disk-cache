## defect inventory

### defect 1: AWS OIDC variable name mismatch

**what**: cloud integration tests fail with "Region is absent" error

**how**: the declapract upgrade changed workflow files to use a different AWS variable:
```diff
- creds-aws-role-arn: ${{ vars.CREDS_CICD_AWS_PREP_OIDC_ROLE_ARN }}
+ creds-aws-role-arn: ${{ vars.CREDS_CICD_AWS_DEV_OIDC_ROLE_ARN }}
```

**why**:
- the declapract practice assumes all repos use `CREDS_CICD_AWS_DEV_OIDC_ROLE_ARN`
- this repo has `CREDS_CICD_AWS_PREP_OIDC_ROLE_ARN` configured
- when the variable does not exist, the AWS auth step is skipped
- without AWS auth, no `AWS_REGION` is set, so the S3 SDK fails

**fix**: reverted the variable name back to `CREDS_CICD_AWS_PREP_OIDC_ROLE_ARN` in:
- `.github/workflows/test.yml`
- `.github/workflows/publish.yml`

## notes

this is a practice bug — the declapract practice should not hardcode a specific AWS variable name. instead, it should allow repos to configure their own variable name or detect which one exists.
